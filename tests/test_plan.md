# Checklist breve de validación

## 1) Checklist de pruebas mínimas
- [ ] El programa corre al menos en un caso exitoso.
- [ ] Existe al menos una prueba de entrada inválida.
- [ ] Se documentó resultado esperado vs resultado obtenido.

## 2) Checklist de documentación
- [ ] `docs/propuesta.md` está completo.
- [ ] `docs/caso_de_uso.md` está completo.
- [ ] `docs/plan_de_pruebas.md` tiene tabla con casos.
- [ ] El alcance mínimo y fuera de alcance están claros.

## 3) Checklist de ejecución
- [ ] `scripts/run.sh` se ejecuta sin errores de sintaxis.
- [ ] El archivo principal está en `src/main.<ext>`.
- [ ] No se usan dependencias complejas ni servicios externos.

## 4) Checklist de entrega final
- [ ] Estructura del repositorio completa y ordenada.
- [ ] Proyecto pequeño y viable.
- [ ] Redacción técnica clara en español.
- [ ] Commit final con cambios limpios.
