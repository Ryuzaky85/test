# Actividad: Diseño de propuesta de práctica temática pequeña

## Descripción general
En esta actividad vas a diseñar una propuesta de práctica pequeña para GitHub Classroom enfocada en arquitectura de computadoras y programación de sistemas. La prioridad es **documentar, justificar y planear** tu idea antes de programar.

## Objetivo de aprendizaje
Al finalizar esta actividad, el estudiante será capaz de:
- Definir un problema concreto y acotado.
- Justificar la elección de un lenguaje para resolverlo.
- Estructurar un repositorio limpio y entendible.
- Documentar caso de uso, alcance y pruebas mínimas.
- Implementar, de forma opcional, un prototipo pequeño y funcional.

## Lenguajes permitidos
- ARM64 Assembly
- C
- Python
- Bash

## Reglas para mantener el proyecto pequeño
1. Debe resolver **un solo caso de uso principal**.
2. Máximo 1 archivo principal en `src/` para el prototipo inicial.
3. Evita interfaces gráficas, frameworks y arquitectura compleja.
4. No se permite usar bases de datos, servicios en la nube ni APIs externas.
5. Si eliges ARM64 Assembly, limita el alcance a programas muy pequeños.
6. Prioriza claridad de documentación sobre cantidad de código.

## Entregables esperados
1. `docs/propuesta.md` completo.
2. `docs/caso_de_uso.md` completo.
3. `docs/estructura_repositorio.md` revisado y, si hace falta, ajustado.
4. `docs/plan_de_pruebas.md` con casos definidos y resultados.
5. `tests/test_plan.md` con checklist final.
6. Prototipo mínimo opcional en `src/main.<ext>`.

## Instrucciones para el estudiante
1. Elige uno de los lenguajes permitidos.
2. Completa primero `docs/propuesta.md`.
3. Después completa `docs/caso_de_uso.md` y `docs/plan_de_pruebas.md`.
4. Ajusta la estructura del repositorio sin romper la base de la actividad.
5. Opcional: implementa un prototipo mínimo en `src/main.<ext>`.
6. Ejecuta `scripts/run.sh` y documenta resultados de prueba.
7. Verifica que tu entrega sea pequeña, viable y entendible.

## Criterios generales de evaluación
- Claridad del problema y viabilidad del alcance.
- Coherencia entre propuesta, caso de uso y plan de pruebas.
- Justificación técnica del lenguaje elegido.
- Orden de la estructura del repositorio.
- Calidad de la redacción técnica.
- Funcionalidad mínima (si se presenta prototipo).

## Ejemplos de posibles temas
- Mini Toolkit en ARM64
- Asistente de Estudio en Terminal
- Reporteador de Información del Sistema
- Organizador de Archivos
- Juego de Aprendizaje en Línea de Comandos

## Nota importante
Primero se documenta la propuesta y su justificación. Después, de manera opcional, se implementa un prototipo pequeño.
